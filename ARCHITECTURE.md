## Ключевые компоненты

### Frontend (Next.js)
- **Порт:** 3000 (внутри Docker), 80/443 (снаружи)
- **Роль:** Обработка UI, OAuth callback, проксирование API
- **Файлы:**
  - `pages/code/callback.js` - обработчик OAuth callback
  - `src/utils/api.js` - API клиент
  - `src/context/AuthContext.js` - управление сессией

### Backend (API)
- **Порт:** 8080
- **Роль:** Бизнес-логика, проверка OAuth кода, управление сессиями
- **Endpoints:**
  - `GET /api/auth/v1/simple/yandex/url` - получить OAuth URL
  - `POST /api/auth/v1/simple/yandex/callback?code=xxx` - проверить код

### Next.js Rewrites
- **Функция:** Прозрачное проксирование API запросов
- **Правило:** `/api/*` → `http://backend:8080/api/*`
- **Преимущество:** Один домен для фронтенда и API

## Безопасность

- ✅ OAuth код передается через query параметр
- ✅ Session cookie хранится в localStorage
- ✅ Все запросы идут через HTTPS
- ✅ Credentials включены в fetch запросах
- ✅ CORS не требуется (один домен)

## Масштабирование

Для масштабирования можно:
1. Добавить Nginx как reverse proxy
2. Запустить несколько инстансов фронтенда
3. Использовать load balancer
4. Добавить Redis для сессий
